MEDIUM CWE-119: ProbeForWrite nur bedingt aufgerufen in FvevolQueryVolumeInfo

Bei Flag=0 (do_not_probe=true) wird Irp->UserBuffer OHNE vorheriges ProbeForWrite beschrieben, obwohl Daten in den User-Buffer kopiert werden. Der Code-Pfad: Irp->UserBuffer (0x70) wird gelesen, OutputBufferLength aus dem IO_STACK_LOCATION (0xb8+0x08) wird als Größe verwendet. ProbeForWrite(UserBuffer, OutputBufferLength, 2) wird NUR aufgerufen wenn das Flag != 0 ist. Danach kopiert FveCopyDataToUserBuffer die Daten. Der interne Bounds-Check in FveCopyDataToUserBuffer (0x14000434c) prüft: (int)(uint)actual_size <= (int)((uint)max_size - (uint)offset) und verhindert so einen Overflow. Ein invalider Pointer führt zum sofortigen Bluescreen (DoS), nicht zur Code Execution. Betroffene Adressen: 0x1400b9b68 (Funktion), 0x1400b9d06 (ProbeForWrite-Call)

Funktion: FvevolQueryVolumeInfo @ 0x1400b9b68
Taint-Path: IOCTL Input -> Irp->UserBuffer -> FveSafeMemcpy
Empfehlung: ProbeForWrite unabhängig vom Flag immer vor UserBuffer-Zugriff aufrufen. Alternativ: Parameter als METHOD_BUFFERED deklarieren, dann validiert der I/O-Manager den Puffer automatisch.
LOW CWE-200: 8-Byte Information Disclosure bei zu kleinem Ausgabepuffer

Wenn der OutputBuffer zu klein ist (Größe < 8 oder tatsachliche DatenGröße > OutputBufferLength), werden TROTZDEM 8 Bytes in den UserBuffer kopiert, bevor STATUS_BUFFER_TOO_SMALL zurückgegeben wird. Code: *puVar2 = *(undefined8 *)puVar4; // Kopiert 8 Bytes Danach: return STATUS_BUFFER_TOO_SMALL; Die 8 Bytes enthalten den Größen-Header der FVE-Datenstruktur -- keine sensitiven Daten wie kryptografische Schlüssel. Dieses Muster ist in Windows-Treibern ublich (Caller erfahrt benötigte Größe), stellt aber technisch eine Informationspreisgabe dar.

Funktion: FvevolQueryVolumeInfo @ 0x1400b9b68
Taint-Path: Internal Buffer -> Irp->UserBuffer (8 bytes, always)
Empfehlung: Nur die benötigte Größe im IoStatus.Information-Feld zurückgeben, KEINE Daten in den UserBuffer kopieren wenn der Puffer zu klein ist.

INFO N/A: Dreifacher Bounds-Check im MDL-I/O-Pfad

BufferSize + Offset < TotalNeeded -> Fehler Alle drei Checks haben Integer-überlauf-Guards. Erst danach wird MmMapLockedPagesSpecifyCache aufgerufen und memcpy ausgeführt.
