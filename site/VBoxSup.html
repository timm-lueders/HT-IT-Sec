MEDIUM CWE-367: IoBuildPartialMdl in IOCTL-Pfad

VBoxSup.sys importiert IoBuildPartialMdl -- klassisches Double-Fetch- Risiko. Zwischen dem ProbeForWrite-Check und der tatsachlichen MDL- Verarbeitung kann ein paralleler Thread den User-Mode-Puffer andern. VirtualBox implementiert jedoch ProbeForRead+ProbeForWrite sowie ExGetPreviousMode was das Risiko reduziert.

Funktion: SUPR0IoCtlPerform @ 0x140019700
Taint-Path: User IOCTL -> ProbeForWrite -> IoBuildPartialMdl -> Kernel Access
Empfehlung: Auf MmProbeAndLockPages + MmMapLockedPagesSpecifyCache migrieren.
LOW CWE-22: Dateisystem-Zugriff aus Kernel-Treiber

VBoxSup.sys kann Dateien uber ZwCreateFile/ZwReadFile offnen (Modul- Loader). Ein bosartiger Pfad konnte zum Lesen beliebiger Kernel- Dateien fuhren. Pfad-Validierung ist kritisch.

Funktion: SUPR0LdrModByName + ZwCreateFile @ 0x1400032d0
Taint-Path: User IOCTL -> Pfad -> ZwCreateFile -> ZwReadFile
Empfehlung: Pfad-Validierung gegen RTL_FILE_ROOT sicherstellen.
